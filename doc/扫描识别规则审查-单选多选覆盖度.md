# 扫描识别规则审查：PDF / TXT / DOCX 单选·多选覆盖度

> 审查对象：`apps/bank-studio/engine/src/document_extractor.cpp`、`rule_based_generator.cpp`、
> `option_label.cpp`。对照真实公务员行测、事业单位公基真题的常见排版做覆盖度评估。
> 日期：2026-07-16。

## 一、结论速览

抽取层（TXT/DOCX/PDF → 纯文本 + 版面锚点）和"选项/答案 token 归一化"这两块做得很扎实：

- 选项 marker 覆盖字母（含全角）、圈码 ①-⑩、带点数字 ⒈、括号数字 `(1)` / 尾括号 `1)`；
- 答案区覆盖：整段"参考答案"、表格答案、`1-5 ABCDE` 区间、题解末尾"故正确答案为 C"叙述式、
  markdown 竖线表格；
- 材料题（共享题干）、跨页统计图、图形选项裁切都有专门处理和测试。

**但"题型判定"这一层有硬缺口，会让一批真题被打回复核或类型判错。** 按严重程度排序如下。

---

## 二、必须补的缺口（会导致真题识别失败）

### 缺口 1 —— 判断题（无 A/B 选项，只有"（ ）"+ 对错）❗最高优先级

真题里判断题极常见，且**通常不列 A/B 选项**，而是：

```
1. C++ 是编译型语言。（  ）
26. 我国宪法规定公民有受教育的权利和义务。（ × ）
```

答案用 `√ / ×`、`对 / 错`、`正确 / 错误`、`T / F`、`是 / 否`。

当前代码（`parseQuestion`）**强制要求至少 2 个选项**，否则报"未识别到至少两个完整选项"打回复核
（`rule_based_generator.cpp:704-705`）。而 `true_false` 类型只在"已经切出正好 2 个选项且文本为
正确/错误/对/错"时才成立（`rule_based_generator.cpp:721-729`）。→ **纯"（ ）"式判断题一律进复核**。

另外 `normalizeAnswer` 不认识 `√ / ×`（`rule_based_generator.cpp:190-218`），即便补了判断题识别，
答案也解析不出来。

**建议：**
1. 新增判断题识别：题干以 `（ ）`/`( )`/`（　）` 结尾且无选项行时，判为 `true_false`，
   自动补 `对/错` 两个选项。
2. `normalizeAnswer` / 判断题答案归一化补 `√→对(A)`、`×→错(B)`、`T/Y/是→对`、`F/N/否→错`。
3. 尾随/括号答案正则（`trailingAnswer`、`inlineAnswerPattern`）补 `√×对错是否TFYN`。

### 缺口 2 —— 多选题靠"章节大标题"标注时全部漏判 ❗

判定多选**只看单题题干是否含"多选题"**三个字：

```cpp
// rule_based_generator.cpp:711
const bool explicitlyMultiple = stemLines.join(...).contains(QStringLiteral("多选题"));
```

但真题的主流排版是**在大标题里标一次，下面每道题干都不再重复**：

```
二、多项选择题（每题2分，多选、错选、少选均不得分）
1. 下列属于……的有：
   A…… B…… C…… D…… E……
2. ……
```

此时每道题的 stem 里都没有"多选题"字样 → 全部按单选处理 → 一旦答案是 `ABD`，就命中
`rule_based_generator.cpp:712-713` 的"题干未标注多选题，却识别到多个答案"→ **整段多选题全被打回复核**。

而且判定词太窄，只认"多选题"，不认这些等价写法：
`多项选择题` / `[多选]` / `（多选）` / `多选` / `不定项` 等。

**建议：**
1. 扫描时识别题型大标题（`isTopLevelSectionHeading` 已能匹配"二、……"），把当前
   section 的题型（单选/多选/不定项/判断）向下传播给该段内的每道题。
2. 放宽判定词：`多选|多项选择|不定项|【多选】|（多选）`。

### 缺口 3 —— 不定项选择题没有独立类型 ❗

不定项选择题（"有一个或多个正确答案"）是事业单位公基固定题型，与多选的**计分规则不同**
（不定项少选得分、多选少选不得分），因此题库类型应当区分。

当前只有 `single_choice / multiple_choice / true_false` 三类，没有 `indefinite_choice`。
不定项若答案是单个字母 → 判成单选；若是多个字母 → 命中"未标注多选题"打回复核。**两种情况都错。**

**建议：** 新增 `indefinite_choice` 类型（需同步 `schemas/`、`bank_validator`），
由缺口 2 的 section 题型传播赋值；对不定项**不做**"答案必须≥2 个"的强校验。

## 三、应补的缺口（少数真题会漏）

### 缺口 4 —— 选项字母范围三处不一致

| 位置 | 支持范围 |
|---|---|
| `option_label.cpp:10-11` `canonicalOptionLabel` | A–I |
| `rule_based_generator.cpp:477` `optionsOnLine` 字母 marker | **A–F** |
| `document_extractor.cpp:193` PDF `optionMarker` 版面锚点 | **A–D** |
| 各答案正则 `[A-Fa-f]{1,6}` | A–F |

后果：含 **E/F 选项的多选题**，PDF 图形选项裁切（依赖 A–D 版面锚点）拿不到第 5、6 个选项；
若真出现 A–G/H（个别多选），`optionsOnLine` 也切不出。建议三处统一到 A–H（覆盖到 8 选项足够），
PDF 锚点至少扩到 A–F。

### 缺口 5 —— 题号/选项"仅空格分隔"不识别

- 题号：`questionPattern`（:38）要求数字后必须跟 `题|.．、:：)）` 分隔符；OCR 常输出
  `1 题干内容`（点被吃掉、只剩空格）→ 不成锚点。
- 选项：`optionsOnLine`（:477）字母 marker 要求分隔符必需（`A.`/`A、`）；OCR 输出 `A 甲 B 乙`
  （只有空格）→ 切不出。

真扫描件（尤其 OCR 回退路径）会踩到。建议对 OCR 来源放宽：允许"字母/数字 + 至少一个空格 +
非空白"作为兜底 marker（需配合"整行至少 2 个 marker"约束，避免误切英文题干）。

### 缺口 6 —— 多选答案的"区间展开"只支持单字母

`globalAnswers` 的 range 分支（:360-391）要求 `末-首+1 == 展开字母数`，只能处理
`1-5 ABCDE` 这类"每题一个字母"。多选答案区 `1-3 AB;BC;ACD` 无法展开（这种写法本就少，
优先级低，记录备查）。

- 圈码/括号数字选项 + 同形态答案 ✅
- 参考答案分区、markdown 表格答案、题解叙述式答案、阶段分组多答案区 ✅
- 材料题（"材料一"/"（一）阅读…"/"根据下列统计资料"）、跨页资料图 ✅
- 页脚 `- 15 -` 剔除、GB18030/UTF-8 TXT 编码回退、DOCX ZIP 正文提取 ✅

| 优先级 | 缺口 | 影响面 | 改动量 | 状态 |
|---|---|---|---|---|
| P0 | 缺口 1 判断题（无选项 + √×） | 判断题整类漏 | 中 | ✅ 已实现 |
| P0 | 缺口 2 多选靠大标题标注 | 多选整段被打回 | 中（需 section 题型传播） | ✅ 已实现 |
| P1 | 缺口 3 不定项独立类型 | 公基固定题型判错 | 中（含 schema/validator） | ✅ 已实现（映射 multiple_choice） |

| P2 | 缺口 5 仅空格分隔（OCR） | 扫描件 | 小-中 | 待办 |

### 已实现说明（2026-07-16）

- **缺口 1 判断题**：`rule_based_generator.cpp` 新增 `hasBlankJudgmentBrackets`（题干以空
  括号“（ ）”结尾）与 `booleanAnswerLabel`（对/错/√/×/是/否 → 合成选项 a/b）。命中时合成
  `id=a 正确 / id=b 不正确` 两个选项并显式定型 `true_false`，答案区扫描新增 `booleanRecord`
  正则支持 `1.√ 2.×` 式判断题答案汇总。**复用现有 true_false 通道，schema/校验器/前端零改动。**
- **缺口 2 多选大标题**：新增 `isMultiAnswerSectionHeading`（识别“多项选择题/不定项”等大标题），
  扫描所有大标题记录“是否多答案”，按题号行之前最近的大标题把 `allowMultipleAnswers` 传播给
  段内每道题。多答案题不再命中“未标注多选却识别到多个答案”复核。
- **缺口 3 不定项**：与多选在作答结构上等价（≥2 选项、≥1 答案），统一映射 `multiple_choice`，
  **不新增题型**（避免动 schema/validator/前端）。区分“严格多选”（题干写“多选/多项选”且非
  不定项，答案<2 才复核）与“不定项/section 传播”（单答案合法，不报“少于两个”）。
- AI 路径提示词（`generation_workflow.cpp`）同步更新，规则与 AI 两条生成路径语义一致。
- 测试：`tests/rule_based_generator_test.cpp` 新增场景 J（判断题 √/×）、场景 K（大标题多选/
  不定项传播），全部 18 个测试通过。

> 缺口 4/5/6 未在本次改动范围内，留待后续。
