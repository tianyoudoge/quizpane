# 小窗刷题：隐蔽模式 UI 与题目图片处理方案

## 1. 结论

技术上可行，并且无需引入 Chromium、OpenCV 或 GPU 强依赖。

当前默认路线是：

- 整个窗口保持透明，只绘制低透明度暗色卡片；
- 题干、选项、作答结果和解析使用低饱和度的明度/色相差异，而不是大字号或鲜艳色块；
- `PIN` 按钮按需切换“始终置顶”，默认开启；
- 上一题、下一题、交卷、尺寸、PIN 和隐藏操作使用无边框单色线性图标；
- UI 提供小、中、大三档，窗口、边距、字号和图标同步缩放并持久化；
- 白底题图采用单次像素扫描的“近白转透明”算法，并裁掉透明边缘；
- 原图不被覆盖，处理结果按图片内容摘要和算法参数缓存；
- 处理失败或内容不适合透明化时允许查看原图。

## 2. 为什么暂不默认启用毛玻璃

真正的毛玻璃需要读取窗口后方内容并模糊。不同系统实现差异较大：

| 平台 | 可选实现 | 第一阶段策略 |
|---|---|---|
| macOS | `NSVisualEffectView` | 可作为设置项增加；默认使用透明暗色卡片 |
| Windows 10/11 | DWM Acrylic/Mica | 可选增强；低端机或远程桌面自动关闭 |
| Windows 7 | DWM Blur Behind，能力有限 | 透明暗色卡片 |
| UOS/Linux | 依赖桌面合成器和窗口管理器 | 透明暗色卡片，不假设一定支持模糊 |

半透明纯色合成只需要窗口合成器完成一次 alpha blending，性能和兼容性都优于实时模糊。毛玻璃应当是可关闭的增强项，不应成为题目可读性的前置条件。

## 3. 字体与信息层级

做题窗口采用三段式结构：标题栏和底部操作栏高度固定，中间答题区在进入练习时根据首题内容计算紧凑高度并锁定。切换到更长题目时窗口与上下栏不跳动，只在答题区显示细滚动条并响应鼠标滚轮。小、中、大三档决定答题区允许使用的最大高度；首题内容较短时不会为了填满档位高度而留下大块空白。

最外层不绘制边框和圆角，并请求系统关闭无边框窗口阴影。分类卡、选项、按钮和滚动条均不使用高对比描边。

暗色透明模式建议使用以下层级：

| 内容 | 颜色方向 | 目的 |
|---|---|---|
| 数据源与页面标题 | 中性浅灰 | 低频导航信息，不形成明显彩色标题 |
| 当前题干 | 暖灰 | 快速定位题目，但避免高饱和黄色 |
| 普通选项 | 浅灰 | 保持长文本阅读舒适 |
| 已选选项 | 轻微明度变化 | 不使用彩色描边和高亮色块 |
| 正确/错误 | 低饱和绿灰/红灰 | 结果反馈，同时保留文字与符号 |
| 解析 | 冷灰绿色和极淡暗底 | 与题干区分，但不形成醒目的模块色块 |

颜色不能作为唯一状态表达方式；正确、错误和选中状态仍保留文字、符号及边框。

## 4. 白底题图处理算法

### 4.1 不采用 Canny 的原因

Canny 适合提取边缘，不适合恢复透明题图。它通常包含高斯滤波、梯度计算、非极大值抑制和滞后阈值连接：

- 至少需要多次遍历和额外缓冲区；
- 会把实心文字、色块和图表转换成轮廓；
- 对低对比度公式和压缩噪声需要反复调参；
- 无法直接得到自然的半透明抗锯齿边缘。

### 4.2 当前算法

实现位置：`quizpane/core/image_privacy_filter.*`。

对每个像素只处理一次：

1. 计算整数近似亮度：`(77R + 150G + 29B) >> 8`；
2. 计算色度：`max(R,G,B) - min(R,G,B)`；
3. 高亮且低色度的像素视为白色背景，转换为透明；
4. 中间亮度区域按线性覆盖率生成 alpha，保留抗锯齿；
5. 色度较高的浅色像素保留，避免浅蓝图表等被误删；
6. 对半透明边缘执行白色底色去污染，避免暗色背景上出现白边；
7. 同一次扫描计算非透明包围盒，最后按需裁掉透明边缘。

默认阈值：

```text
完全透明亮度：248
完全不透明亮度：218
彩色保护色度：18
裁剪 alpha 阈值：8
裁剪留白：6 px
```

### 4.3 性能与内存

- 时间复杂度：`O(width × height)`；
- 扫描次数：1 次，裁剪时由 Qt 复制最终有效区域；
- 不依赖 OpenCV、GPU、神经网络或平台指令集；
- 1920×1080 ARGB 图像约占 8 MiB；处理过程中通常保留输入和输出两份；
- 当前 macOS Debug 构建中，含 1920×1080 测试的整项测试约为几十毫秒量级；Release 版本会更快；
- 正式接入时应在解码线程执行，并按内容摘要缓存结果，UI 线程只接收完成后的 `QImage`。

### 4.4 降级和边界情况

以下图片不应强制透明化：

- 白色本身代表数据的热力图、医学图、地图；
- 摄影图片或带纹理的浅色背景；
- 透明化后有效信息面积异常小；
- 用户手动选择“查看原图”。

Provider 或 Host 可根据图片类型声明默认策略。UI 至少提供“透明图/原图”切换，不能不可逆地修改缓存源文件。

## 5. 后续接入顺序

1. 从题目模型中分离远程图片资源，不让富文本标签直接任意加载网络内容；
2. Provider 下载图片并执行尺寸、MIME、解码像素上限检查；
3. Host 在线程池解码和执行近白转透明；
4. 按 `图片摘要 + 阈值参数 + 算法版本` 缓存；
5. UI 限制最大显示尺寸，保留滚轮和点击查看原图；
6. 根据真实题图增加按题型禁用、阈值微调和失败自动降级。
