# 题库生成器 UI 与小窗刷题集成方案

## 产品边界

题库生成器是独立 Qt Widgets 应用，窗口名为“题库生成器 · 小窗刷题”。它与小窗刷题共享仓库、Qt 版本、颜色和控件语言，但使用正常桌面窗口尺寸，不使用悬浮小窗。

它同时承担 Agent Host 和工作流编排器的角色。用户只需安装图形化生成器，不需要安装 Codex、Claude Code、Python、Node.js、Docker 或任何模型命令行工具。

拆分成独立进程的原因：

- PDF/DOCX 解析、OCR 和模型调用可能占用较多内存，不能影响刷题窗口；
- 生成任务通常持续数分钟，适合正常窗口而非便签尺寸；
- 生成器只输出标准 `.quizpane-provider`，主程序只负责安装，边界稳定；
- 两个应用可以独立升级、崩溃隔离，也便于以后单独分发生成器。

## 非开发者向导

生成任务固定为四步；模型属于应用级设置，不占用向导步骤：

1. **选择资料**：拖入 TXT、Markdown、DOCX、PDF 或 JSON；
2. **自动整理**：显示当前阶段、进度和 Token 用量；
3. **检查问题**：只展示缺少答案、疑似重复、选项错位等异常；
4. **完成**：填写题库名称和每套题数量，生成跨平台安装包。

模型在菜单 `设置 → 模型设置…` 中统一管理，不占用向导步骤。普通用户直接选择 OpenAI、Anthropic Claude、DeepSeek、阿里云百炼、智谱 AI 或 Ollama；只有选择“自定义供应商（高级）”后才需要手工填写 Endpoint。普通用户不会看到 Manifest、ABI、Schema 或动态库。

## Token 与费用呈现

进度页区分三个数字：

- 输入 Token（估算）：本地分段后、请求发送前的估算值；
- 输出 Token：只能使用模型 API 返回的真实 usage，不能用字符数伪装；
- 本次合计：实际输入与输出之和，流式生成时持续累计。

模型未调用时必须显示输出 Token 为 0，并明确说明没有上传资料。价格信息由独立模型目录提供；用户自定义兼容服务只展示 Token，不擅自估算费用。

## 进度模型

总体进度由确定性阶段权重计算，而不是无限循环动画：

| 阶段 | 权重 |
|---|---:|
| 文件读取与安全检查 | 5% |
| 文字、表格和图片提取 | 20% |
| 分段与 Token 预算 | 5% |
| 模型结构化 | 40% |
| Schema 与语义校验 | 15% |
| 定向修复和去重 | 10% |
| 打包与自检 | 5% |

每个分块保存状态，暂停、异常退出或网络中断后可以从检查点恢复。取消请求不删除本地提取结果。

## 模型选择

### 厂商目录

生成器内置以下厂商目录：

| 供应商 | Endpoint 策略 | 模型列表策略 |
|---|---|---|
| OpenAI | 固定官方 API 地址 | 使用 API Key 请求 `/models`，失败时加载内置推荐列表 |
| Anthropic Claude | 固定官方 API 地址 | 使用 Anthropic 请求头请求 `/models`，失败时加载内置推荐列表 |
| DeepSeek | 固定官方兼容地址 | 请求 `/models`，失败时加载内置推荐列表 |
| 阿里云百炼 / 通义千问 | 固定官方兼容地址 | 请求 `/models`，失败时加载内置推荐列表 |
| 智谱 AI | 固定官方兼容地址 | 请求 `/models`，失败时加载内置推荐列表 |
| Ollama | 固定本机地址 | 请求 `/api/tags` 读取已经安装的本地模型 |
| 自定义供应商（高级） | 用户填写 OpenAI-compatible Endpoint | 请求 `<Endpoint>/models`，失败时允许手工填写模型名 |

固定厂商的 Endpoint 只读，避免普通用户误改。每个厂商在下拉框中带轻量标识，并提供可点击的 API Key 注册入口和官方配置教程。获取模型列表只发送鉴权信息，不上传用户导入的题目资料；网络、权限或返回格式异常时，界面会说明原因并自动回退到内置模型名。

自定义供应商用于公司网关、代理服务、私有部署和其他 OpenAI-compatible 服务。高级用户需要自行保证 Endpoint、鉴权方式和模型兼容性。

API Key 最终应进入系统安全凭据库，不写入项目文件、日志或题库包。当前 UI 骨架只在生成器进程内存中保留本次填写的 Key；在真实模型调用接入前，需要复用主程序的跨平台安全凭据层完成持久化。

模型调用、分段并发、失败重试、定向修复、Token 统计和打包都在生成器进程内部完成。模型只是远程或本地推理服务，不承担工作流控制，也不要求用户理解 Agent 概念。

## 与小窗刷题的连接

小窗刷题菜单路径：`题库管理 → 创建题库…`。点击后启动独立生成器进程。

查找顺序：

1. 开发/测试环境变量 `QUIZPANE_BANK_STUDIO`；
2. 与主程序同一安装位置；
3. macOS `/Applications/题库生成器.app`；
4. Linux `PATH`。

生成成功后，生成器应调用主程序文件关联打开 `.quizpane-provider`，由小窗刷题显示最终安装确认。两个进程不共享裸指针、模型密钥或未校验的题目数据。
