# 题库生成器 UI 与小窗刷题集成方案

## 产品边界

题库生成器是独立 Qt Widgets 应用，窗口名为“题库生成器 · 小窗刷题”。它与小窗刷题共享仓库、Qt 版本、颜色和控件语言，但使用正常桌面窗口尺寸，不使用悬浮小窗。

拆分成独立进程的原因：

- PDF/DOCX 解析、OCR 和模型调用可能占用较多内存，不能影响刷题窗口；
- 生成任务通常持续数分钟，适合正常窗口而非便签尺寸；
- 生成器只输出标准 `.quizpane-provider`，主程序只负责安装，边界稳定；
- 两个应用可以独立升级、崩溃隔离，也便于以后单独分发生成器。

## 非开发者向导

界面固定为五步：

1. **选择资料**：拖入 TXT、Markdown、DOCX、PDF 或 JSON；
2. **选择模型**：图形化选择 OpenAI、OpenAI 兼容服务或 Ollama；
3. **自动整理**：显示当前阶段、进度和 Token 用量；
4. **检查问题**：只展示缺少答案、疑似重复、选项错位等异常；
5. **完成**：填写题库名称和每套题数量，生成跨平台安装包。

普通用户不会看到 Manifest、ABI、Schema 或动态库。高级模型地址和名称可以修改，但默认值可直接使用。

## Token 与费用呈现

进度页区分三个数字：

- 输入 Token（估算）：本地分段后、请求发送前的估算值；
- 输出 Token：只能使用模型 API 返回的真实 usage，不能用字符数伪装；
- 本次合计：实际输入与输出之和，流式生成时持续累计。

模型未调用时必须显示输出 Token 为 0，并明确说明没有上传资料。价格信息由独立模型目录提供；用户自定义兼容服务只展示 Token，不擅自估算费用。

## 进度模型

总体进度由确定性阶段权重计算，而不是无限循环动画：

| 阶段 | 权重 |
|---|---:|
| 文件读取与安全检查 | 5% |
| 文字、表格和图片提取 | 20% |
| 分段与 Token 预算 | 5% |
| 模型结构化 | 40% |
| Schema 与语义校验 | 15% |
| 定向修复和去重 | 10% |
| 打包与自检 | 5% |

每个分块保存状态，暂停、异常退出或网络中断后可以从检查点恢复。取消请求不删除本地提取结果。

## 模型选择

MVP 模型适配器：

- OpenAI Responses/Chat；
- OpenAI-compatible endpoint；
- Ollama 的本地 OpenAI-compatible endpoint。

API Key 进入系统安全凭据库，不写入项目文件、日志或题库包。点击“测试连接”只发送最小健康检查，不上传用户文档。

## 与小窗刷题的连接

小窗刷题菜单路径：`题库管理 → 创建题库…`。点击后启动独立生成器进程。

查找顺序：

1. 开发/测试环境变量 `QUIZPANE_BANK_STUDIO`；
2. 与主程序同一安装位置；
3. macOS `/Applications/题库生成器.app`；
4. Linux `PATH`。

生成成功后，生成器应调用主程序文件关联打开 `.quizpane-provider`，由小窗刷题显示最终安装确认。两个进程不共享裸指针、模型密钥或未校验的题目数据。
