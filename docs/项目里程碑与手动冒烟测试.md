# QuizPane 项目里程碑与手动冒烟测试

> 适用基线：`dev` 分支 `4cc7285`（QuizPane 0.2.4）
> 目标平台：Windows 10 x64、macOS Apple Silicon
> 文档口径：“已实现”表示代码和自动测试链路已存在；“已验收”必须有真实发行包和目标系统的手工证据。

## 1. 产品交付主线

QuizPane 由两个独立桌面程序和一个共享运行时组成：

```text
用户原始题库（TXT / Markdown / DOCX / PDF）
       │
       ▼
题库制作器 Question Maker
  ├─ 本地文档提取（扫描 PDF 走 Tesseract OCR）
  ├─ 规则结构化（离线）或 OpenAI-compatible 模型生成
  ├─ 共享材料 / 题组 / 来源页码 / 待复核标记
  ├─ BankValidator 统一校验
  └─ 生成 .quizpane-provider
       │
       ▼
小窗刷题 QuizPane Desktop
  ├─ ProviderInstaller 安全解包
  ├─ DeclarativeProvider 加载题库
  ├─ 分类 / 组卷 / 答题 / 交卷 / 解析
  └─ 草稿恢复 / 材料题展示 / 题库管理
```

核心技术栈为 C++20、Qt 6 Widgets/Network/Pdf、CMake/Ninja、Tesseract C++ OCR、miniz 题库包，以 GitHub Actions 生成 Windows、macOS 和 Linux 发行产物。

## 2. 已完成的关键里程碑

| 里程碑 | 当前状态 | 已经具备的能力 | 仍需验收的风险 |
|---|---|---|---|
| K0 跨平台工程骨架 | 已实现 | CMake target 分层，Windows/macOS/Linux 构建脚本，两个独立应用包 | 需用真实 Actions 产物验证无缺库、无架构错配 |
| K1 Provider 统一契约 | 已实现 | 唯一 `schemaVersion: 2`，支持独立题、共享材料、`materialId`、来源页码与复核元数据 | 需用用户题库验证边缘格式不会生成错误引用 |
| K2 安全安装与运行 | 已实现 | `.quizpane-provider` 检查、解包大小限制、路径安全、声明式 Provider 加载 | Windows 权限、中文路径、长路径和 macOS 首次启动需真机测试 |
| K3 刷题主程序闭环 | 已实现 | 分类、组卷、单选/判断、草稿、交卷、解析、材料卡、题库切换 | 用户题库的长文本、长选项、材料题跨页展示需人工确认 |
| K4 文档提取 | 已实现 | UTF-8/GB18030 TXT、Markdown、DOCX OOXML、文字 PDF；PDF 保留页边界 | DOCX 复杂表格/文本框、PDF 双栏与乱序文字属已知风险 |
| K5 扫描 PDF OCR | 已实现 | Tesseract C++ 后端默认编译和打包，中英文训练数据随包分发 | 需验证 Win10/macOS 包能找到 tessdata，并记录实际识别率与速度 |
| K6 离线规则结构化 | 已实现 | 题号、A–F 选项、行内/文末答案、解析、材料范围；不确定内容进入复核 | 不规整编号、多选题、图片题和复杂排版不能保证自动通过 |
| K7 模型生成 | 已实现 | OpenAI-compatible 请求、语义切块、单次定向修复、Token 统计、检查点恢复 | Anthropic/Gemini 原生协议、并发请求和密钥持久化未完成 |
| K8 人工复核与打包 | 已实现 | 异常聚合展示，待复核题默认不入包，可按题/题组取舍，打包前再走统一校验 | 当前更偏“采纳/丢弃”，不是完整题目编辑器 |
| K9 可诊断性 | 已实现 | DEBUG 包日志、菜单“查看调试日志”、5 MiB 轮转、敏感字段脱敏、崩溃栈/Windows dump | 需用真实崩溃样本确认符号和 PDB 能还原到有用栈帧 |
| K10 发行体积控制 | 已实现 | Release 剔除符号；macOS ZIP level 9；Windows Optimal ZIP；Linux gzip-9/xz-9e | 应在每次发行记录两个应用的压缩包与解压体积，设置回归阈值 |

### 当前最重要的判断

1. 主干数据链已经贯通：原始文档 → 结构化 → 校验 → Provider → 安装 → 答题。
2. 产品当前的主要风险已从“缺少主链路”转为“输入格式的长尾兼容性”和“发行包真机可用性”。
3. 下一个发布门槛不应是继续堆功能，而应是 Win10 + Apple Silicon 的真实题库冒烟通过。

## 3. 后续交付里程碑

### M1：发行稳定化（当前最高优先级）

准入条件：当前 `dev` 生成 Windows x64 和 macOS arm64 产物。

- 完成本文第 5 节的双平台手工冒烟。
- 固定一套可公开的 TXT/DOCX/文字 PDF/扫描 PDF 回归语料。
- 记录包体、首启时间、OCR 速度、题目识别率和峰值内存。
- 修正真机发现的缺库、tessdata 定位、中文路径和权限问题。
- 接入 Apple Developer ID 签名/公证，Windows 根据分发规模决定代码签名。

完成定义：P0/P1 冒烟用例全部通过，无崩溃、无黑屏、无缺少动态库，同一 Provider 在两个平台的题数、题组关系和答案一致。

### M2：复核和题型能力

- 把复核页从“采纳/丢弃”升级为可编辑题干、选项、答案、解析和材料归属。
- 端到端支持多选题：Schema、制作器、Provider RPC、答题 UI、草稿、判分和解析同步上线。
- 再评估填空/简答题；在判分语义未明确前不抢跑扩展 Schema。

### M3：复杂文档理解

- 为双栏 PDF、表格、图文混排引入“布局块”中间层，不再只传递纯文本。
- 支持题干图、选项图和材料图的提取、裁剪、去白底、打包和展示。
- 给 OCR 增加页级质量评分与人工重试入口，而不是对低质量结果静默继续。

### M4：模型与安全

- API Key 进入 macOS Keychain / Windows Credential Manager，日志和检查点严禁持久化密钥。
- 补充 Anthropic Messages 和 Gemini GenerateContent 原生适配。
- 在保证检查点与输出可重放后，引入有上限的并发分块、指数退避和速率限制。

### M5：题库生态与更多平台

- 官方题库索引/市场、发布者签名、权限展示和可回滚更新。
- 补齐题库数据完全删除、凭据清理和版本迁移。
- Linux/UOS/麒麟真机认证，再根据用户需求评估 ARM64、LoongArch64 等架构。

## 4. 手动冒烟的准备方式

### 4.1 测试机

| 代号 | 环境 | 必填信息 |
|---|---|---|
| W10 | Windows 10 22H2 x64，尽量使用未安装 Qt/Tesseract/开发工具的普通用户环境 | OS build、CPU、内存、剩余磁盘、系统语言 |
| MAS | Apple Silicon Mac（M1–M4），建议当前受支持 macOS | macOS 版本、芯片、内存、剩余磁盘、系统语言 |

每台机器都应使用同一次 CI 的产物和同一批源文件。不要用开发机本地 build 目录中的程序替代发行包。

### 4.2 建议准备的原始题库样本

| 样本 ID | 内容要求 | 主要验证点 |
|---|---|---|
| S01-TXT | UTF-8 TXT，20 道规整单选/判断，含题内答案和文末答案表 | 基本规则解析 |
| S02-GB | GB18030 TXT，文件名与路径含中文和空格 | 编码和路径兼容 |
| S03-DOCX | Word 题库，含段落、简单表格、粗体/换行、15–50 题 | OOXML 提取顺序 |
| S04-PDF-TEXT | 可选中文字的 PDF，至少 10 页，题目与答案跨页 | Qt PDF 提取与页码 |
| S05-PDF-OCR | 300 DPI 左右的中文扫描 PDF，3–10 页，可混少量英文 | Tesseract/tessdata 随包和 OCR 精度 |
| S06-MATERIAL | 2 篇阅读/资料材料，每篇 3–5 道子题，加 2 道独立题 | 材料边界与 `materialId` |
| S07-DIRTY | 故意缺答案、重复题、选项错位、多选答案、不支持扩展名 | 失败语义和待复核默认排除 |
| S08-LARGE | 接近真实上限的大文档，建议 100–300 页副本 | 长时任务、暂停/恢复、内存和日志体积；不属于 P0 冒烟 |

测试资料中如有真实用户内容，不得提交 Git，不得上传公开 issue。模型模式会把文本发送到所选服务；敏感题库只使用离线规则模式。

### 4.3 证据要求

每个失败用例至少保留：

- 测试平台、包名、SHA-256、测试时间；
- 源文件的脱敏名称、格式、大小、页数；
- 实际结果、预期结果、屏幕截图；
- DEBUG 包对应日志与崩溃产物，必要时再补 VERBOSE 日志。

DEBUG 日志默认位置：

- Windows：`%LOCALAPPDATA%\QuizPane\logs\`
- macOS：`~/Library/Application Support/QuizPane/logs/`

主程序和题库制作器分别生成自己的 `*-debug.log`；Windows 崩溃产物为 `*-crash.dmp`，macOS 为 `*-crash.log`。Release 包不启用该调试日志菜单。

## 6. 通过标准与测试记录

### 6.1 发布门槛

- W10 和 MAS 的所有 P0 用例必须通过。
- P1 失败必须有明确的 issue、影响范围和发布决策，不允许口头遗留。
- 任一平台的崩溃、题库无法安装、OCR 后端缺失、答案错位、草稿丢失，均视为 P0。
- 跨平台同一 Provider 的题数、材料数、关联关系和答案必须一致。
